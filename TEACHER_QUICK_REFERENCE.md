# Teacher Quick Reference Guide

## 🚀 Quick Actions

### Create an Exam
```
Exams Tab → New Exam → Fill Details → Add Questions → Publish
```

### Assign Students
```
Exams Tab → Assign Button → Select Students → Save
```

### Grade Subjective Questions
```
Exams Tab → Results → Grade Button → Enter Marks → Save Grades
```

### Add Image to Question
```
Create Question → Add Image → Upload → Drag/Zoom to Crop → Apply
```

### Notify Students
```
Exams Tab → Notify Button → Confirm
```

## 📊 Understanding Results

### Trust Score
- **80-100%**: Excellent integrity, no concerns
- **50-79%**: Some violations detected, review logs
- **0-49%**: Multiple violations, likely cheating

### Status Indicators
- **Passed**: Score ≥ 40% of total marks
- **Failed**: Score < 40% of total marks
- **Disqualified**: Exceeded violation tolerance limit

## 🤖 AI Question Generation

### By Topic
1. Click "AI Generate" in exam creation
2. Select "By Topic" mode
3. Enter topic (e.g., "Binary Search Trees")
4. Choose type, count, difficulty
5. Click "Generate"

### From Document
1. Click "AI Generate" in exam creation
2. Select "From Document/Syllabus" mode
3. Paste syllabus or lecture notes (min 50 chars)
4. Choose type, count, difficulty
5. Click "Generate from Doc"

## 🔒 Security Settings

### Webcam
- **ON**: Students must enable camera (recommended)
- **OFF**: No camera requirement

### Browser Lock
- **ON**: Prevents tab switching (strict mode)
- **OFF**: Allows tab switching (lenient mode)

### Tolerance Limit
- **1**: Strict - terminates after 1 violation
- **3**: Standard - terminates after 3 violations (recommended)
- **5**: Lenient - terminates after 5 violations

## 📝 Question Types

### MCQ (Multiple Choice)
- Single correct answer
- Auto-graded
- Supports negative marking

### Multiple Correct
- Multiple correct answers
- Auto-graded
- Supports negative marking

### Short Answer
- Text response
- Requires manual grading
- Can provide model answer for reference

### Coding
- Code response
- Requires manual grading
- Can provide model solution for reference

## 🎯 Best Practices

### Before Exam
1. ✅ Create exam at least 24 hours in advance
2. ✅ Assign specific students (don't leave unassigned)
3. ✅ Test exam by viewing as student
4. ✅ Send notification 1 day before exam
5. ✅ Set appropriate tolerance limit

### During Exam
1. ✅ Monitor Results tab for live submissions
2. ✅ Check trust scores in real-time
3. ✅ Review violation logs for suspicious activity

### After Exam
1. ✅ Grade subjective questions within 48 hours
2. ✅ Review low trust score submissions
3. ✅ Export results for record-keeping
4. ✅ Provide feedback to students

## 🆘 Common Issues

### Students can't see exam
- **Solution**: Check if students are assigned to the exam

### AI generation fails
- **Solution**: Ensure topic is clear or document has 50+ characters

### Image won't crop
- **Solution**: Try uploading a smaller image (< 5MB)

### Grading not saving
- **Solution**: Ensure marks don't exceed maximum marks for question

### Trust score seems wrong
- **Solution**: Review violation logs to see what was detected
